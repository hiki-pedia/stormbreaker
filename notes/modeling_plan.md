# 모델링 계획

## 1. 기본 방향

이번 문제는 2025년 시간별 `kpx_group_1`, `kpx_group_2`, `kpx_group_3` 발전량을 예측하는 회귀 문제다.

핵심 입력은 평가 기간에도 제공되는 `LDAPS/GFS 기상예보 데이터`다. SCADA와 실제 발전량은 학습 구간에서만 제공되므로, 2025년 추론 단계에서 직접 입력 피처로 사용하지 않는다.

가장 중요한 원칙은 다음과 같다.

- 원본 데이터는 수정하지 않는다.
- 랜덤 분할이 아니라 시간 기준 검증을 사용한다.
- 2025년 타깃 기간의 실제 발전량, SCADA, 실제 기상 관측값은 사용하지 않는다.
- 외부 기상청 관측 데이터는 우선순위가 낮은 추가 실험으로 두며, 누수 가능성을 엄격히 통제한다.
- 예측값은 각 그룹 설비용량 범위로 후처리한다.

## 2. 검증 전략

리더보드는 2025년 평가 데이터로 계산되지만, 우리는 정답을 알 수 없다. 따라서 내부 검증셋을 2025년과 최대한 비슷한 방식으로 만들어야 한다.

기본 검증 방식:

- `kpx_group_1`, `kpx_group_2`
  - 학습: 2022-2023년
  - 검증: 2024년
- `kpx_group_3`
  - 학습: 2023년
  - 검증: 2024년

`kpx_group_3`는 2022년 정답이 없기 때문에 2022년을 학습에 사용하지 않는다.

검증 지표:

- 그룹별 NMAE
- 평균 NMAE
- 1-NMAE
- 가능하면 FiCR 근사 지표

공개 리더보드는 일부 샘플이므로, 공개 리더보드 점수만 보고 모델을 선택하지 않는다. 내부 검증 점수와 리더보드 점수를 함께 기록한다.

## 3. 1차 주력 알고리즘: LightGBM

첫 주력 모델은 `LightGBM`으로 한다.

사용 이유:

- 표 형태 데이터에 강하다.
- LDAPS/GFS처럼 컬럼이 많은 수치형 피처에 잘 맞는다.
- 비선형 관계를 자동으로 학습할 수 있다.
- 풍속과 발전량의 포화 관계, 계절성, 격자별 차이를 트리 구조로 잘 잡을 수 있다.
- 학습 속도가 빠르다.
- GPU 없이 MacBook에서도 충분히 실험 가능하다.
- 결측값이 있어도 기본적으로 처리할 수 있다.
- 피처 중요도를 확인하기 쉬워서 이후 전략 수립에 도움이 된다.

기본 학습 방식:

- 그룹별 별도 모델 3개를 학습한다.
- 입력 피처는 LDAPS/GFS 기반 피처와 시간 피처를 사용한다.
- 타깃은 각 그룹의 시간별 발전량이다.
- 예측값은 `0`과 그룹 설비용량 사이로 범위 제한한다.

## 4. 보조 알고리즘 후보

### CatBoost

LightGBM 다음으로 비교할 모델이다.

사용 이유:

- 작은 데이터와 노이즈가 있는 데이터에서 안정적인 경우가 많다.
- 기본 파라미터 성능이 좋은 편이다.
- 과적합 제어가 비교적 편하다.
- LightGBM과 예측 경향이 달라 앙상블에 도움이 될 수 있다.

### XGBoost

LightGBM/CatBoost와 비교용으로 사용한다.

사용 이유:

- 트리 부스팅 계열의 강한 기준 모델이다.
- 튜닝이 잘 되면 안정적인 성능을 낸다.
- LightGBM과 다른 방식의 정규화 효과를 기대할 수 있다.

### ExtraTrees / RandomForest

주력 모델은 아니고 앙상블 후보로 둔다.

사용 이유:

- 부스팅 모델과 다른 오류 패턴을 만들 수 있다.
- 과도한 튜닝 없이 기준 모델 다양성을 확보할 수 있다.
- 예측 분산을 줄이는 앙상블 재료로 쓸 수 있다.

### 딥러닝 모델

초기 우선순위는 낮다.

이유:

- 데이터 크기가 딥러닝이 압도적으로 유리한 형태는 아니다.
- 핵심은 대규모 모델보다 피처 설계, 검증, 후처리다.
- 리더보드 실험 속도와 재현성을 고려하면 트리 기반 모델이 먼저다.

## 5. 핵심 피처 설계

### 기상 피처

- U/V 바람 성분
- 풍속: `sqrt(u^2 + v^2)`
- 풍향: `atan2(v, u)`
- 풍향의 `sin`, `cos`
- 풍속 제곱, 풍속 세제곱
- 격자별 평균, 최솟값, 최댓값, 표준편차
- LDAPS와 GFS의 풍속 차이
- 고도별 풍속 차이
- 기온, 습도, 기압, 구름, 강수 관련 변수

### 시간 피처

- 월
- 시간
- 계절
- 연중 일자
- 연속 시간 인덱스
- 월/시간의 `sin`, `cos`

### 공간 피처

`info.xlsx`의 터빈 좌표를 이용한다.

- 그룹별 가까운 LDAPS 격자 선택
- 그룹별 거리 가중 평균
- 그룹별 주변 격자 평균
- 그룹 1, 2, 3을 같은 격자 평균으로만 처리하지 않기

현재 확인한 대략적인 가까운 LDAPS 격자:

- 그룹 1: LDAPS 5, 6, 10 근처
- 그룹 2: LDAPS 6, 11, 7 근처
- 그룹 3: LDAPS 12, 11, 7 근처

GFS는 격자가 넓어서 세 그룹 모두 비슷한 GFS 격자 영향을 받을 가능성이 크다.

## 6. SCADA 활용 방침

SCADA는 학습 구간에서만 제공된다. 따라서 2025년 추론 입력으로 직접 사용하지 않는다.

사용 가능한 방향:

- 터빈별 풍속-출력 곡선 이해
- VESTAS와 UNISON의 발전 특성 비교
- 그룹별 출력 포화 구간 파악
- 예보 풍속이 발전량으로 바뀌는 비선형 관계를 이해
- 피처 아이디어와 후처리 아이디어 도출

주의할 점:

- 2025년 SCADA는 제공되지 않는다.
- SCADA 기반 피처가 추론 시점에도 만들 수 있는 형태인지 확인해야 한다.
- 추론 시 만들 수 없는 피처는 모델 입력으로 넣지 않는다.

## 7. 외부 기상청 데이터 사용 전략

기상청 기상자료개방포털에는 ASOS/AWS 등 지상 관측자료가 있다.

- ASOS 자료: https://data.kma.go.kr/data/grnd/selectAsosRltmList.do
- AWS 자료: https://data.kma.go.kr/data/grnd/selectAwsRltmList.do

다만 실제 관측자료는 누수 위험이 크다. 특히 2025년 평가 기간의 실제 관측값은 목표 시각의 실제 날씨를 알려주므로, 그대로 사용하면 예측 문제가 쉬워지고 규칙 위반 소지가 생긴다.

따라서 기본 방침은 다음과 같다.

- 1차 모델에서는 외부 기상청 데이터를 사용하지 않는다.
- 제공 데이터만으로 강한 기준 모델을 먼저 만든다.
- 외부 데이터는 별도 실험으로 분리한다.
- 2025년 타깃 기간의 실제 기상 관측값은 사용하지 않는다.
- 외부 데이터를 쓰더라도 2022-2024년 학습 구간의 관계 분석, 보정 아이디어 검토 용도로 우선 사용한다.
- 외부 데이터 사용 시 출처, 수집 기간, 수집 시점, 전처리 코드를 모두 기록한다.

가능한 외부 데이터 활용 예시:

- 태백 인근 ASOS/AWS 관측소의 과거 풍속/풍향/기온 분포 확인
- LDAPS/GFS 예보와 실제 관측값의 과거 편향 분석
- 예보 풍속 보정 아이디어 도출
- 단, 2025년 평가 구간 실제 관측값으로 직접 피처를 만들지 않는다.

## 8. 실험 순서

### exp001: 기상예보만 사용한 LightGBM 기준 모델
일단 LDAPS/GFS 기상예보만 평균내서 기본 모델을 만든다.(기본모델)

- 데이터: LDAPS/GFS
- 피처: 기본 기상 변수, 풍속, 시간 피처, 전체 격자 집계
- 모델: 그룹별 LightGBM
- 목적: 재현 가능한 첫 기준점 확보

### exp002: 공간 피처 추가
터빈 위치와 가까운 기상 격자를 더 중요하게 써본다.(위치반영)

- 그룹별 가까운 LDAPS 격자
- 거리 가중 평균
- 그룹별 주변 격자 통계
- 목적: 터빈 위치와 격자 위치 정보를 반영

### exp003: 풍력 물리 피처 추가
풍속, 풍향, 풍속², 풍속³ 같은 풍력에 중요한 물리 피처를 더 강화한다.(풍력 물리 반영)

- 풍속 제곱
- 풍속 세제곱
- 풍향 sin/cos
- 고도별 풍속 차이
- 목적: 풍속-발전량의 비선형 관계 반영

### exp004: 모델 비교
LightGBM 말고 CatBoost, XGBoost도 돌려서 어떤 모델이 더 잘 맞는지 비교한다.(모델비교)

- LightGBM
- CatBoost
- XGBoost
- 목적: 모델별 내부 검증 성능 비교

### exp005: 앙상블
잘 나온 여러 모델의 예측을 섞어서 더 안정적인 예측을 만든다.(모델섞기)

- LightGBM + CatBoost
- LightGBM + XGBoost
- 그룹별 다른 모델 선택
- 목적: 예측 안정화

### exp006: 후처리 최적화
예측값을 설비용량 범위에 맞게 다듬고, 그룹별 보정계수로 점수를 끌어올린다.(제출값 보정)

- 설비용량 기준 범위 제한
- 그룹별 보정계수
- 낮은 발전량 구간 보정
- FiCR 관점의 임계값 조정
- 목적: 평가식에 맞춘 점수 개선

### exp007: 외부 기상청 데이터 검토
나중에 기상청 외부 데이터를 누수 없이 쓸 수 있는지 검토해본다.(외부 기상 데이터 검토)

- 2022-2024년 태백 인근 ASOS/AWS 관측자료 검토
- 2025년 타깃 기간 실제 관측값은 제외
- 제공 예보와 실제 관측의 과거 편향 분석
- 기준 모델 대비 개선 여부를 별도 검증
