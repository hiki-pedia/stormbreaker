# 결측치 처리 방침

## 1. 기본 원칙

결측치 처리는 점수 향상보다 먼저 누수 방지를 기준으로 결정한다.

- 원본 `train/`, `test/` 파일은 수정하지 않는다.
- 결측 처리는 피처 생성 코드 안에서 재현 가능하게 수행한다.
- 타깃 결측은 임의로 채우지 않는다.
- 평가 기간의 미래 정보를 이용한 보간은 하지 않는다.
- 모델이 자연스럽게 처리할 수 있는 결측은 우선 `NaN`으로 남기는 방식을 허용한다.

## 2. 타깃 결측 처리

대상 파일: `train/train_labels.csv`

### 결정

타깃 결측 행은 해당 그룹 모델 학습에서 제외한다.

예:

- `kpx_group_1` 모델 학습 시 `kpx_group_1`이 결측인 행 제외
- `kpx_group_2` 모델 학습 시 `kpx_group_2`가 결측인 행 제외
- `kpx_group_3` 모델 학습 시 `kpx_group_3`가 결측인 행 제외

### 이유

타깃은 우리가 맞혀야 하는 정답이다. 정답 결측을 평균, 보간, 다른 그룹 값 등으로 채우면 가짜 정답을 학습하게 된다.

특히 `kpx_group_3`는 2022년 정답이 없으므로, `kpx_group_3` 모델은 2023년 이후 데이터만 사용한다.

## 3. LDAPS/GFS 기상 입력 결측 처리

### 학습 데이터

확인 결과 `train/ldaps_train.csv`, `train/gfs_train.csv`에는 주요 결측이 없다.

### 평가 데이터

`test/gfs_test.csv`는 결측이 없다.

`test/ldaps_test.csv`에는 일부 시각과 일부 변수에서 결측이 있다.

결측 발생 시각:

- `2025-04-08 17:00:00`
- `2025-06-18 18:00:00`
- `2025-07-18 06:00:00`

### 1차 방침

LightGBM 기준 모델에서는 LDAPS 결측을 `NaN`으로 남긴다.

이유:

- LightGBM은 결측값을 자체적으로 분기 처리할 수 있다.
- 결측 규모가 작다.
- 임의 보간보다 누수 위험이 낮다.
- 첫 기준 모델에서는 단순하고 재현 가능한 처리가 더 중요하다.

### 2차 방침

NaN 처리만으로 성능이나 안정성이 부족하면, 다음 순서로 실험한다.

1. 학습 데이터 기준 변수별 중앙값 대체
2. 학습 데이터 기준 격자, 월, 시간별 중앙값 대체
3. 같은 시각의 GFS 유사 변수로 보조 대체

단, 평가 기간의 앞뒤 시각을 사용한 양방향 보간은 기본적으로 사용하지 않는다. 특정 예측 시점 이후에 공개되는 정보를 끌어올 가능성이 있기 때문이다.

## 4. 파생 피처 결측 처리

풍속, 풍향, 풍속 제곱, 풍속 세제곱 등 파생 피처는 원천 변수가 결측이면 `NaN`으로 둔다.

예:

```text
u 또는 v 결측 → 풍속 결측
u 또는 v 결측 → 풍향 sin/cos 결측
```

이후 모델이 `NaN`을 처리하게 한다.

단, 선형모델이나 결측을 허용하지 않는 모델을 실험할 경우에는 학습 데이터 기준 통계값으로 대체한다.

## 5. SCADA 결측 처리

SCADA는 2025년 추론 시 직접 사용할 수 없으므로, 기준 모델 입력에는 넣지 않는다.

따라서 초기 실험에서는 SCADA 결측을 모델 입력 결측 문제로 다루지 않는다.

SCADA를 사용할 때의 방침:

- SCADA는 풍속-출력 관계 분석, 그룹별 특성 파악, 후처리 아이디어 도출에 우선 사용한다.
- SCADA에서 발전량/풍속/풍향이 결측인 행은 해당 분석에서 제외한다.
- SCADA 결측을 보간해서 2025년 입력 피처처럼 쓰지 않는다.
- 추론 시 만들 수 없는 SCADA 기반 피처는 모델 입력으로 넣지 않는다.

## 6. 제출값 결측 처리

최종 제출 파일에는 결측값이 있으면 안 된다.

모델 예측 후 다음 처리를 적용한다.

1. 예측값이 `NaN`이면 해당 그룹의 학습 데이터 기준 동일 월, 시간 중앙값으로 대체한다.
2. 그래도 `NaN`이면 해당 그룹의 학습 전체 중앙값으로 대체한다.
3. 예측값을 그룹별 설비용량 범위로 제한한다.

설비용량 기준:

- `kpx_group_1`: `0` ~ `21600`
- `kpx_group_2`: `0` ~ `21600`
- `kpx_group_3`: `0` ~ `21000`

## 7. 실험 기록

결측 처리 방식이 바뀌면 반드시 `notes/experiment_log.md`에 기록한다.

기록할 항목:

- 어떤 컬럼/피처의 결측을 처리했는지
- 어떤 기준으로 대체했는지
- 학습 데이터 기준 통계만 사용했는지
- 평가 데이터의 미래 시점을 사용하지 않았는지
- 검증 점수 변화

## 8. 현재 결정 요약

초기 기준 모델 결측 처리:

- 타깃 결측: 해당 그룹 학습에서 제외
- 학습 기상 데이터: 별도 처리 없음
- 평가 LDAPS 일부 결측: LightGBM에는 `NaN` 유지
- 파생 피처 결측: `NaN` 유지
- SCADA 결측: 기준 모델 입력에서 제외
- 제출 예측 결측: 학습 기준 월, 시간 중앙값으로 대체 처리
