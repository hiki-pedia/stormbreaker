# 풍력발전량 예측 AI 경진대회 (BARAM 2026) - 진행 기록

마지막 업데이트: 2026-07-14

## 대회 개요

- 대회: 제3회 풍력발전량 예측 AI 경진대회 (dacon, https://dacon.io/competitions/official/236727)
- 목표: 2025년 전체(8,760시간)에 대해 KPX 그룹 1/2/3의 시간별 발전량(kWh) 예측
- 평가 산식: **총점 = 0.5 × (1-NMAE) + 0.5 × FICR**
  - NMAE: 그룹별 `평균(|예측-실제| / 설비용량)`, 실제 발전량이 설비용량의 10% 이상인 시간대만 평가
  - FICR(정산금획득률): 시간대별 nMAE 구간에 따라 정산 단가 차등 적용
    - nMAE 6% 이하 → 4원/kWh, 6~8% → 3원/kWh, **8% 초과 → 0원**
    - FICR = 획득 정산금 / (전 시간대가 6% 이하였을 때의 이론상 최대 정산금)
- 일정: 대회 시작 7/6, 종료 8/14, 오프라인 발표평가 8/28
- 규칙: data leakage 금지(예측기준시점 이후 정보 사용 불가 - 사후 실측/보정자료/재분석자료 등), Python만 사용, 사전학습 모델은 2026-07-05 이전 공개 오픈소스만 가능, 원격 API 모델 금지

## 데이터 구조

- `train/ldaps_train.csv`, `test/ldaps_test.csv`: LDAPS 기상예보 (1.5km 해상도, 16개 grid_id)
- `train/gfs_train.csv`, `test/gfs_test.csv`: GFS 기상예보 (0.25도 해상도, 9개 grid_id)
- `train/train_labels.csv`: 실제 발전량 (kpx_group_1/2/3, kWh). group_3는 2023년부터만 존재(2022년 결측)
- `train/scada_vestas_train.csv`, `train/scada_unison_train.csv`: 터빈별 SCADA 실측 (10분 단위). **test 기간엔 없음 (사용 불가)**
- `info.xlsx`: 터빈 메타정보
- `sample_submission.csv`: 제출 양식

## 핵심 발견 (EDA)

1. **터빈-그룹 매핑** (info.xlsx로 확인, 전부 태백가덕산 풍력단지):
   - `kpx_group_1` = VESTAS 1~6호기 (3.6MW×6 = 21.6MW)
   - `kpx_group_2` = VESTAS 7~12호기 (3.6MW×6 = 21.6MW)
   - `kpx_group_3` = UNISON 1~5호기 (4.2MW×5 = 21.0MW)
2. **SCADA 검증**: 그룹별 터빈 SCADA power 합산 시 실제 라벨과 상관계수 0.997~0.9998로 거의 완벽 일치. 단, 일부 컬럼에 ±4천만 수준 극단 이상치가 0.05% 섞여 있어 전처리 필요 (정격출력 기준 클리핑).
3. **풍속-발전량 상관**: 터빈에서 가장 가까운 LDAPS grid(11번, ~0.5km)의 10m 풍속 상관계수 0.73 > GFS 최근접 grid(~5.5km) 0.62. LDAPS가 고해상도·근접성 이점 있음.
4. **계절성/일중패턴**: 겨울철 발전량↑, 여름철↓. 설비이용률 평균 26~33%.
5. **기후평균(단순 월×시간대 평균) 베이스라인 NMAE ≈ 0.256** vs 학습모델 NMAE ≈ 0.13~0.15 — 모델이 실질적 신호를 잡고 있음을 확인.

## 코드 구조

- `src/features.py`: LDAPS/GFS 원자료 → forecast_kst_dtm 기준 집계 피처
  - 풍속(U/V→크기), 풍향(sin/cos), 공기밀도 보정 플럭스(rho×ws³), 격자간 mean/std/max/min 집계
  - `add_lag_features()`: 과거 방향 랙(t-1,t-2)/이동평균 피처 (미래 시점은 예보 블록 경계에서 누출 위험 있어 의도적으로 제외)
- `src/train_baseline.py`: 그룹별 LightGBM 회귀 파이프라인
  - `load_scada_group_wind()`, `add_physics_features()`: SCADA 실측 풍속 기반 캘리브레이션+파워커브 피처 (물리 기반, 실효성은 낮았음)
  - `nmae()`, `ficr()`, `total_score()`, `error_bin_distribution()`: 대회 공식 평가지표를 그대로 구현한 로컬 검증 함수
  - `build_walk_forward_folds()`: 연도 단위 expanding-window 검증 (2022→2023, 2022+2023→2024 등). 표본 1000행 미만인 연도(라벨 마지막 시각 잔여분)는 제외
  - `run_group()`: num_leaves 탐색 + 배깅(5시드) + 커스텀 손실함수 지원
  - `search_blend_weights()`: LDAPS-only/GFS-only/combined 3개 변형 모델의 블렌딩 가중치를 대회 총점 기준으로 탐색
  - `next_submission_path()`: 실행할 때마다 `submission_baseline_v{N}.csv`로 자동 버전 증가 저장

## 실제 리더보드 결과 (제출 1회차)

- 순위: 500명 중 422등
- 총점 0.61169 = 1-NMAE 0.87225 + FICR 0.35113 (0.5:0.5 가중)
- 로컬 검증(0.86대)과 리더보드 1-NMAE(0.87225)가 상당히 근접 — 로컬 검증 방법론 자체는 신뢰할 만함을 확인

## 핵심 병목: FICR

- 리더보드 기준 NMAE는 준수하나 FICR이 0.35 수준으로 낮음
- 원인: 평균 NMAE가 12~15%인데 정산 기준 임계값은 6~8% → 시간대의 **57~64%가 "8% 초과, 정산금 0원" 구간**에 위치
- v5(배깅), v6(임계값 가중 손실함수) 등 다양한 방식으로 이 병목을 공략했으나 **셋 다 0.588~0.590 범위로 수렴** → 현재 피처/데이터로 뽑아낼 수 있는 신호의 한계에 도달한 것으로 판단

## 시도했지만 효과 없었던 것들 (다시 안 해도 됨)

- 풍향(sin/cos) 피처: 피처 중요도엔 포함되지만 전체 NMAE 개선은 미미
- 공기밀도 보정 플럭스(rho×ws³): 피처 중요도 1~2위 차지했지만 기존 풍속 피처와 정보 중복 → 순위만 대체, 실질 개선 없음
- 랙/이동평균 피처: 소폭 기여하나 임팩트 작음 (이미 격자간 std/max/min이 변동성 포착)
- SCADA 기반 물리 파워커브: 단독 성능이 오히려 트리모델보다 나쁨 (NMAE 0.186~0.19) - 캘리브레이션에 피처 1개만 사용해 트리모델(300+ 피처)보다 정보량 부족
- 배깅(5시드 평균): 병목이 모델 분산이 아니라 예보 자체의 오차라 효과 없음
- 정산 임계값 가중 손실함수: 이론적으로는 맞는 방향이었으나 예보 오차 자체가 임계값보다 큰 게 근본 원인이라 손실함수 교체만으로는 한계

## 다음에 시도해볼 만한 것 (미시도, 잠재력 있음)

1. **완전히 다른/추가 공개 데이터** — 대회 규정상 허용되는 공개 재분석자료 등. 지금까지 시도가 전부 기존 피처의 재조합 수준이라 한계에 부딪힘. 새로운 정보원이 있어야 다음 단계 도약 가능해 보임.
2. **16개 LDAPS 격자의 공간 구조(4x4)를 그대로 보존하는 모델** (CNN 등) — 지금은 mean/std/max/min으로 뭉개서 공간 패턴(풍향 일관성, 지형 구배 등) 정보 손실 가능성. 다만 데이터 양(3년치)이 적어 성공 불확실.
3. FICR 정산금 기준표는 스크린샷으로 확보함 (6%/8% 임계값, 4원/3원/0원) — `train_baseline.py`의 `SETTLEMENT_TIERS`에 반영되어 있음.
